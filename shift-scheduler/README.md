# 排班工具設計說明

## 目標

提供一個**輕量、免登入**的排班工具，可管理人員、班別與每日班表，並以月曆檢視與匯出。

## 功能範圍

| 功能 | 說明 |
|------|------|
| **人員管理** | 新增／編輯／刪除成員，每人可設名稱與選填備註 |
| **班別設定** | 自訂班別名稱與顏色（如：早班、晚班、休假、請假） |
| **月曆排班** | 以月為單位顯示日曆，點選日期＋人員可指派班別 |
| **資料持久化** | 使用瀏覽器 localStorage，關閉後再開仍保留 |
| **班表條件** | 設定每週幾（週一～週日）各需要幾人上哪一種班（如每週一 2 人早班、3 人晚班）；月曆上會顯示「已排/需求」提示 |
| **匯出** | 可匯出當月或指定範圍班表為 JSON／簡易文字，方便備份或列印 |

## 技術選擇

- **純前端**：單一 HTML + CSS + JavaScript，無需後端或資料庫。
- **localStorage**：資料存於本機，同一瀏覽器、同一網域下持久保存。
- **無框架**：方便直接放在現有專案中，用瀏覽器開檔或掛在任意靜態站即可。

## 資料結構（簡要）

```json
{
  "staff": [
    { "id": "uuid", "name": "王小明", "note": "" }
  ],
  "shiftTypes": [
    { "id": "uuid", "name": "早班", "color": "#3b82f6" },
    { "id": "uuid", "name": "晚班", "color": "#ef4444" }
  ],
  "assignments": {
    "2025-03-08": {
      "staffId": "shiftTypeId"
    }
  },
  "weeklyRequirements": {
    "1": { "shiftTypeId": 2 },
    "2": { "shiftTypeId": 3, "otherShiftId": 1 }
  }
}
```

- `assignments` 的 key 為日期字串 `YYYY-MM-DD`，value 為「人員 id → 班別 id」的對應。
- `weeklyRequirements` 的 key 為星期幾（0＝週日、1＝週一 … 6＝週六），value 為「班別 id → 需要人數」的對應。

## 使用方式

1. **本機直接開啟**：用瀏覽器開啟 `index.html` 即可使用。
2. **與現有網站整合**：可將 `index.html` 放到靜態路徑，或複製內容到書籍轉換工具專案中的一個新路由（若需要同一網域下共用）。

## 後續可擴充

- 匯出為 CSV／Excel 供列印或統計。
- 每人每週最少／最多班數檢查。

- 若日後需要多裝置同步，可改為後端 API + 資料庫。
